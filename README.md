# Restaurant_order_SQL
Proyek ini bertujuan untuk menganalisis data pemesanan restoran menggunakan teknik SQL mulai dari level dasar hingga intermediate. Analisis ini memberikan gambaran mendalam mengenai performa penjualan, menu terlaris, perilaku pelanggan, segmentasi berdasarkan RFM, serta pattern operasional restoran seperti jam sibuk dan metode pembayaran favorit.
.

# Objectives

Proyek ini bertujuan untuk:

Mengidentifikasi pola penjualan harian dan jam operasional paling ramai

Menentukan menu dan kategori yang paling menguntungkan

Menganalisis perilaku pelanggan berdasarkan total transaksi dan frekuensi order

Membuat segmentasi pelanggan dengan metode RFM Analysis

Memberikan insight bisnis melalui data kuantitatif

Menunjukkan kemampuan SQL untuk keperluan portofolio profesional

# Database Schema

Schema: restaurant_portfolio
Table: orders

Struktur umum tabel orders:

Column Name	Description
order_id	ID transaksi
order_date	Tanggal pemesanan
order_time	Waktu pemesanan
customer_id	ID pelanggan
menu_item	Nama menu
category	Kategori menu (drinks, appetizer, dll.)
quantity	Jumlah item yang dibeli
